## 2026-03-31 - Fix useDeferredValue dependency trap\n\n**Learning:** Passing an entire object into a `useMemo` dependency array defeats `useDeferredValue` if another property on that object changes frequently (like a search input query triggering object recreation). The `useMemo` re-runs synchronously on every keystroke with the _old_ deferred value, causing input lag.\n**Action:** Destructure the needed primitive properties from the state object and use those in the dependency array alongside the deferred value.
